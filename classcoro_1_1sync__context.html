Detailed Description

helps synchronization between two threads when future is used

Causes that writer is blocked until reader completes processing the data. You need to use future::wait or future::get() with instance of this object.

To use this object, simply declare the variable on stack. Variable is not copyable nor movable

Note
this object can't simulate coroutine by a thread if the resume is called in the same thead as wait(). In this case, the thread cannot block the caller, and the thread continues in execution once the caller finishes its work. In such case it is better to use callbacks
