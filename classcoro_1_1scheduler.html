◆ scheduler()

coro::scheduler::scheduler ( )
default

initialize the scheduler. It is not started, you must start it manually

The scheduler in this state can be used for scheduling, however no awaiting coroutine is awaken until the scheduler is started

◆ block()

blocked coro::scheduler::block ( ident_t  ident)
inline

block specified identity to be scheduled

Parameters
identidentity to block.
Returns
object which must be held to block identity

The function cancels awaiting sleep operation if any, and blocks all futher sleep operations for given identity. This block is held during lifetime of returned object

Blocked identity cannot be used for scheduling and attempt to sleep with it ends with the exception await_canceled_exception()
