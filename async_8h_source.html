13  //Helps to handle symmetric transfer while coroutine is already destroyed
14  /*
15  * HACK
16  *
17  * Reason: MSC has a bug, which prevents to use symmetric transfer when coroutine frame is destroyed inside of await_suspend of the final
18  * suspend operation. The function must be finished while frame is still valid. However we need to destroy the frame before transfering to
19  * next coroutine. So this class implements transfering coroutine, which handles this situation
20  *
21  *
22  * it is inifnite and empty generator. It is thread local allocated. You just need to call prepare_jump to set jump handle, and destroy handle.
23  * Return value is handle to jump;
24  *
25  * The generator's promise_type handles destruction of previous coroutine frame and jump to next coroutine.
26  *
27  * There is actually double jump
28  *
29  * (finished_coro) -> SymmTransGen -> (coroutine to resume)
30  *
31  * while frame of finished coroutine is destroyed in process
32  *
33  * NOTE: There is no promise that this is effective fast solution. It only garantee that
34  * this solution performs true symmetric transfer (with no extra frame allocation).
35  * Please ask Microsoft to fix this bug
36  */
